git安装配置

Git是一种分布式版本控制和源码管理系统，不论是个人写代码还是团队协作，git都能给我们的开发带来很大的便利。而github就是一个最流行的git存取站点，而且它还提供了社交化软件开发功能。

git

我们将使用git作为drupal模块开发过程中的版本控制工具，因为drupal官方就是用git做源码管理，我们在开发使用git,会方便我们以后将模块提交给drupal.

git安装


yum install git
apt-get install git
http://msysgit.github.io

git配置

在安装完git以后，要进行的基本配置有下面几个：
1.配置用户信息
git config —global user.name “your name”
git config —global user.email yourname@example.com

2.颜色配置，这个在windows的效果不是很好
git config —global color.ui auto

3.配置文件的作用域
/etc/gitconfig 对所有用户有效
~/.gitconfig 对当前用户有效
{工作目录}/.git/config 仅对当前项目有效

$HOME 或者 C:Documents and Settings$USER 下的.gitconfig
git安装目录下的/etc/gitconfig

4.配置要忽略的文件（.gitignore）,往往在项目中我们有许多文件不需要进行版本控制，像有些临时文件，图片文件，缓存文件等.

  • gitignore文件格式:
  • gitignore的作用范围
    — 全局有效的配置

做完上边的配置之后，我们要在home目录建一个.gitigore文件，这个文件将对所有项目有效。如果项目需要上传到git服务器，gitignore的配置不会上传。

— 局部有效的配置
直接在项目目录下建立.gitignore文件即可，同时这个文件也会被git跟踪，并跟随项目保存到服务器，也可以在这个文件中加入一行.gitignore，将其本身忽略。

— 注意
对于已经commit的文件，gitignore不会对它们起作用，需要先从版本库中删除，再添加。

第一个drupal模块

目标

我们的第一模块将实现以下功能：通过一个区块，展示出已经启用的模块列表。
我们将分三步实现这个完成这个模块：

  • 建立模块目录和模块文件
  • 建立区别
  • 利用drupal中的SimpleTest框架写一个自动测试

按照敏捷开发的思想，我们应该先写测试，但是为了简单起见，我们采用了上面的流程.
不管怎么样，我们的目标是说明如何写一个模块，而不是演示具体的开发方法。我们这样做会更简单一些，这主要有两个原因：

  • SimpleTest是我们这一章最简单的一部分，它的代码量是实际模块的两倍。
  • 我们在尝试写测试的时候，应该先熟悉这些在开发中会用到的api。

在正规的模块开发过程中，你可以遵循TDD的开发流程，先写测试，然后写模块。但是我们现在以学习为主，所以还是按照我们上边的流程，先来写我们的模块。

写一个新模块

建立一个新的drupal模块是很简单的，有多么简单呢？简单到现在已经有超过5000个drupal模块，甚至许多drupal模块的开发者都是php新手。其实，接下来我们这个模块的代码就可以说明开发一个模块是多么容易，完成一个模块只需要一个目录和两个小文件。

模块的名字

一个drupal模块需要两个名字：
  人读名字:这个名字在.info 文件中用”name”去定义的，可以是一个或几个单词，单词的首字母大写，多个单词用空格分开。
  机读名字:这个名字就是模块目录的名字，可以包括大、小写字母，数字和下划线。

这两个名字应该尽量一致，比如“Administration menu”模块的机读名字就是“admin_menu”。
另外说明一点为了避免模块重名，你最好事先检查一下，你选用的模块“机读名称”是否已经存在了，你可在https://www.drupal.org/project/ 这个网址的后面加上你的模块的机读名称，看看官网上是否已经存在同名的文件。

模块放在哪


/modules 这个目录下边存放的是系统的核心模块，你不能改动这个目录下的任何文件，系统升级时有可能从写这个目录。
/sites/all/modules 我们安装的第三方的扩展模块会放到这个目录下边，drush默认会把模块下载到这个目录。
/sites/default/modules 这是推荐的放置我们模块的目录。modules目录默认是不存在的，需要手动创建。放到这里的好处是能够把我们自己的代码和标准的扩展模块区分开，便于查找和管理。放到这里的一个不利因素是，如果是在多网站的环境下，这个目录下边的模块将只对default这个站点起作用。

模块目录

由于这是我们的第一个模块，我们就把它的机读名称命名为”frist”,接下来建立一个新的目录/sites/default/ modules/first，然后新建一个first.info文件和first.module文件，我们的模块就只需要这两个文件。
下面我们就开始写.info文件。

建立.info文件

这个文件的作用是向drupal提供一些有关这个模块的基本信息，例如：模块的名称，依赖的其它模块，提供的代码文件等等。
.info文件是一个类似于ini配置文件的纯文本文件,每一行由‘名称’，‘等号’和‘值’组成，按照drupal的编码规范，等号的两边各有一个空格。
有些键名可以有多个值，这时候需要采用类似数组的定义方式，它的格式就像下边这样：
name[] = value1
name[] = value2
如果“值”的内容超过一行，要使用引号引起来。
以分号（;）开头的行会被当做注释处理。
好了，下面我们就看一下first.info的完整内容

建立.module文件

一般一来说，.module文件中包含了所有主要的勾子实现。

规范我们的代码

下面是一些需要特别注意的drupal代码规范：

  • 缩进
  • <?php ?>
  • 运算符两边空格
  • 流程控制
  • 函数
  • 变量

    doxygen代码风格区块

注释以斜线（/）和两个星号()开始，以一个星号（）和一个斜线（/）结束，其它每一行以星号开始，这种风格的注释被称为“文档块”（ documentation block）。@file 表示这是整个文件的描述，而不是文件中某一部分的描述，所有文件都应该以文档级别的文档块开始。
在@file下边一行是对本文件的简单介绍，一般在长度在一行之内。随后需要隔一行进行详细的打述

help hook

drupa中定义了一个有关“帮助”的勾子，通过实现这个勾子，可以在用户访问帮助系统时，显示这个模块的帮助信息

一个函数是如何成为一个勾子的实现的呢？就是靠它的名字： first_help(). 这个名字遵循了勾子的模式。如果一个勾子命名为hook_help,那么要实现它，只需把“hook”这个词替换成模块的名字。因此，要实现hook_help这个勾子，只要用first模块的名字来定义一个函数first_hook().
每一个勾子都自己的参数，所有的drupal核心提供的勾子文档都被整理在http://api.drupal.org.
hook_help() 需要两个参数

  • $path: 帮助系统的url路径
  • $arg : 当访问这个url的时候，用到的参数
    在我们这个例子中，只涉及到第一个参数。基本上，帮助系统就是通过url路径匹配帮助文本。我们的模块需要定义给这引起特定的url提供什么样的文字信息。当访问admin/help#MODULE_NAME的时候，在模块中定义的帮助信息就会显示出来，其中MODULE_NAME指的是模块的机器名称。
    这个函数是通过检查$path来做出响应的，如果$path设置的是admin/help#first,那么它将返回一些简单的帮助信息。

drupal的开发者应该理解的t函数的三个特性：

  • 当t函数被调用时发生了什么
  • drupal是怎么维护翻译表的

drush

drush是一个关于drupal的命令行工具，它可以让我们通过命令行的方式更方便的完成页面操作。最简单的比如：下载模块，清除缓存，修改主题等等。

drush的安装

drush有多种安装方式，我们这里只介绍windows下的安装，如果大家想了解drush的其它方式或详细情况，可以访问它的官网，或者github

  • 到http://www.drush.org/drush_windows_installer下载drush安装包.
  • 像其它的windows程序的安装一样，直接下一步就可以完成安装。
  • 在安装完成以后可以通过Drush Command Prompt来运行drush命令，如果想在命令行运行drush需要将drush的目录注册到系统的环境变量当中。

    drush的常用命令

    drush有很多功能可以简化我们的操作，下面只是列出一些经常用到的，具体的大家可以到官网查阅。

drupal7模块开发--概述

drupal从2000年发展至今，已经成为非常成熟和完善的开源框架，它不仅有着极其丰富的功能框架和模板，更加重要的是除了软件本身，它的社区和运作机制已经非常完善。在这个过程中，drupal也变得越来越庞大和复杂。使学习者要想掌握它，需要学习的东西也越来越多。下面我我们就来梳理一下drupal当中最重要的部分有哪些：

为什么要学习drupal模块开发？

  1. 在学习过程中可以提高自己的技术水平。
    通过不断深入接触drupal能够使我们对它的设计思想有更深入的了解，而且它的模块包括的范围非常广泛，我们能够接触到drupal之外的许多新技术，像webservers, redis缓存， 安全认证等等，都在drupal的模块中有所提现。
    而且drupal有着非常规范的代码标准，这也能使我们在学习过程中，使自己写的代码更加规范，在模块开发过程中我们还会应用过版本控件，功能测试等良好的开发习惯。

  2. 可以使我们更加灵活高效的应对用户的各种需求。
    drupal的模块虽然已经非常的多，已经能够满足我们大多数的功能需求，可是这并能保证我们的项目，可以完全用别人的模块来完成，而且很多时候即使我们可以使用第三方的模块，但这并不一定能达到最佳的效果。

  3. 真正了解开源软件的意义。
    开源并不是简单的开放源代码，而且一种“开放”， “共享”， “协作”的运作模式，除了代码，人们之间如何相互协作也是值得我们学习和思考的。
    我们不仅是从社区获取需要的模块和帮助，我们也是社区的贡献者。我们可以从提交bug, 问题反馈，回答问题，提供补丁和代码的过程中，真正融入到开源社区，体现自身价值。

当然，drupal本身也存在诸多问题，但这也正好给了我们去完善它的机会，如果你对开源软件感兴趣，可以尝试一下drupal.
